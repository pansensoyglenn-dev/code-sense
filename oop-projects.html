📚 LIBRARY MANAGEMENT

Complete library system with books, members, loans, and fines. Demonstrates private attributes with name mangling, property decorators for controlled access, method resolution order with multiple inheritance, and mixins for timestamp functionality. Full-featured catalog with search, hold, renewal capabilities, and fine calculation for overdue items. Includes specialized collections for reference materials, periodicals, and digital media with different borrowing rules.

class LibraryItem:
    def __init__(self, title, isbn):
        self.__title = title
        self._isbn = isbn
        self.checked_out = False
        self._hold_count = 0

class Book(LibraryItem):
    def __init__(self, title, isbn, author, pages):
        super().__init__(title, isbn)
        self.author = author
        self.pages = pages
        self.genre = None

class ReferenceBook(Book, TimestampMixin):
    def __init__(self, title, isbn, author, pages):
        super().__init__(title, isbn, author, pages)
        self.in_library_use_only = True

💰 BANKING SYSTEM abstract classes · composition · polymorphism

Transaction log, multiple account types, overdraft protection, interest strategies. Uses abstract base classes to enforce interface contracts and method overriding for specialized behavior. Supports checking, savings, and money market accounts with different fee structures and minimum balances. Demonstrates composition through transaction history objects and account holder relationships. Implements the decorator pattern for adding features like overdraft protection and interest calculation.

from abc import ABC, abstractmethod

class Account(ABC):
    def __init__(self, owner):
        self.owner = owner
        self._balance = 0
        self.transactions = []
    
    @abstractmethod
    def calculate_interest(self):
        pass

class SavingsAccount(Account):
    def __init__(self, owner, interest_rate=0.02):
        super().__init__(owner)
        self.interest_rate = interest_rate
    
    def calculate_interest(self):
        return self._balance * self.interest_rate

🛒 STRATEGY PATTERN open/closed · dependency injection · composition

Multiple discount strategies injected into shopping cart. Perfect example of composition over inheritance and the open/closed principle. Supports percentage discounts, fixed amount off, buy-one-get-one-free, loyalty rewards, and seasonal promotions. Strategies can be combined and stacked. Demonstrates how to design for change by encapsulating varying behavior behind a common interface.

class DiscountStrategy(ABC):
    @abstractmethod
    def apply(self, cart):
        pass

class PercentageDiscount(DiscountStrategy):
    def __init__(self, percentage):
        self.percentage = percentage
    
    def apply(self, cart):
        return cart.subtotal * (1 - self.percentage/100)

class Cart:
    def checkout(self, strategy):
        return strategy.apply(self)

📊 OBSERVER PATTERN event-driven · pub/sub · loose coupling

Stock price notifier with multiple subscribers. Demonstrates one-to-many dependency and loose coupling through the observer pattern. Includes logging observers for audit trails, alert observers for threshold notifications, analytics observers for real-time statistics, and dashboard observers for visualization. Implements both push and pull models of data transfer. Perfect for event-driven architectures.

class Subject:
    def __init__(self):
        self._observers = []
    
    def attach(self, observer):
        self._observers.append(observer)
    
    def notify(self, *args, **kwargs):
        for observer in self._observers:
            observer.update(self, *args, **kwargs)

class Observer(ABC):
    @abstractmethod
    def update(self, subject, *args, **kwargs):
        pass

🏭 FACTORY PATTERN creational · encapsulation · SRP

Document factory creating different document types (PDF, Word, HTML, Markdown, JSON, XML). Each document type has its own renderer, exporter, and format-specific configurations. Demonstrates factory method pattern for flexible object creation and abstract factory for families of related objects. Encapsulates creation logic following the Single Responsibility Principle. Makes adding new document types trivial without modifying existing code.

class Document(ABC):
    @abstractmethod
    def render(self):
        pass
    
    @abstractmethod
    def export(self):
        pass

class DocumentFactory:
    @staticmethod
    def create(document_type, title, author):
        factories = {
            'pdf': PDFDocument,
            'html': HTMLDocument,
            'markdown': MarkdownDocument
        }
        return factories[document_type](title, author)
