PYTHON FUNDAMENTALS
A complete reference — from first principles to practical patterns
01 · Core Language
what is Python?
Python is a high-level, interpreted, general-purpose language created by Guido van Rossum (1991). It emphasises readability — developers express ideas in fewer lines than Java or C++. Python runs cross-platform (Windows, macOS, Linux) and powers web backends, data science, AI/ML, automation, and scripting. Its motto: "batteries included" — a vast standard library ships with every install.
dynamic & strongly typed
Python uses dynamic typing — types are inferred at runtime, no declarations needed. But it is strongly typed: mixing incompatible types (like "2" + 2) raises a TypeError instead of silently coercing. This gives prototyping speed without hidden bugs. Inspect types at runtime with type(x) or isinstance(x, int).
# Type inspection x = 42 print(type(x)) # <class 'int'> print(isinstance(x, int)) # True print(x + " apples") # TypeError!
variables & assignment
Variables are created on first assignment. Python supports multiple assignment, tuple unpacking, and augmented assignment (+=, -=, *=, etc.). Names are case-sensitive and use snake_case by convention. Constants are written in ALL_CAPS (convention only — Python doesn't enforce them).
name = "Alice" # str age = 30 # int pi = 3.14159 # float active = True # bool empty = None # NoneType a, b, c = 1, 2, 3 # tuple unpacking a, b = b, a # swap — no temp variable needed score += 10 # augmented assignment
indentation & scope
Blocks are defined by indentation (4 spaces — never mix tabs and spaces). No curly braces. Every indent level opens a new scope. Python resolves names using the LEGB rule: Local → Enclosing → Global → Built-in. Use global x or nonlocal x to write to outer scopes.
def greet(name): if name: print(f"Hello, {name}!") else: print("Hello, world!") greet("Glenn") # Hello, Glenn! greet("") # Hello, world!
02 · Data Types
numbers
Three numeric types: int (arbitrary precision, no overflow), float (IEEE 754 double), complex (3+4j). Key operators: // floor division, ** exponentiation, % modulo. The math module adds sqrt, log, ceil, floor, sin, and more.
print(10 // 3) # 3 floor division print(10 % 3) # 1 modulo print(2 ** 10) # 1024 power print(abs(-7)) # 7 print(round(3.14159, 2)) # 3.14 import math print(math.sqrt(144)) # 12.0 print(math.pi) # 3.14159...
strings
Strings are immutable sequences of Unicode characters. Use single, double, or triple quotes. f-strings (Python 3.6+) are the preferred interpolation method. Strings support indexing, slicing, concatenation (+), and repetition (*). Key methods: .upper(), .lower(), .strip(), .split(), .join(), .replace(), .startswith(), .find().
s = "Python" print(s[0]) # P (first char) print(s[-1]) # n (last char) print(s[1:4]) # yth (slice) print(s[::-1]) # nohtyP (reversed) print(s.upper()) # PYTHON print(" hi ".strip()) # "hi" name, ver = "Glenn", 3.12 msg = f"Hello {name}, Python {ver:.1f}"
built-in data structures
Python's four core collections power almost every program:
  • list — ordered, mutable, duplicates OK. Use for sequences.
  • tuple — ordered, immutable. Use for fixed records, dict keys.
  • dict — key→value mapping, O(1) lookup. Ordered since Python 3.7.
  • set — unique values, O(1) membership test, supports union/intersection.
# List fruits = ["apple", "mango", "guava"] fruits.append("lemon") fruits.sort() print(fruits[0]) # apple # Dict person = {"name": "Glenn", "age": 25} person["city"] = "Manila" print(person.get("city", "unknown")) # Set tags = {"python", "code", "python"} print(tags) # {'python', 'code'}
booleans & comparisons
True and False are boolean literals (capital). Logical operators are English words: and, or, not. Python supports chained comparisons like 0 < x < 10. Falsy values: 0, 0.0, "", [], {}, set(), None, False. Use is / is not to compare identity (required for None checks).
print(5 > 3) # True print(5 == 5.0) # True (value equality) print(not False) # True print(True and False) # False print(1 < 2 < 3) # True (chained) x = None print(x is None) # True (always use 'is' for None)
03 · Control Flow
if / elif / else
Conditional logic uses if, elif (not "else if"), and else. Python also supports a compact ternary expression: x if condition else y. Since Python 3.10, the match statement adds powerful structural pattern matching similar to switch-case.
score = 85 if score >= 90: grade = "A" elif score >= 75: grade = "B" else: grade = "C" # Ternary (one-liner) status = "pass" if score >= 60 else "fail" print(grade, status) # B pass
for loops
Python's for iterates over any iterable — lists, strings, dicts, ranges, generators. Use range(start, stop, step) for numeric sequences. enumerate() gives index + value. zip() pairs two iterables. break exits early; continue skips to next iteration.
for i in range(5): print(i, end=" ") # 0 1 2 3 4 fruits = ["apple", "mango", "guava"] for i, fruit in enumerate(fruits): print(f"{i}: {fruit}") names = ["Ana", "Bob"] scores = [95, 88] for name, sc in zip(names, scores): print(f"{name}: {sc}")
while loops
while repeats while a condition is true. Python loops can have an optional else clause — it runs when the loop finishes without hitting break. This is useful for search patterns: the else means "item was not found."
n = 1 while n <= 5: print(n, end=" ") n += 1 # 1 2 3 4 5 # for/else — "not found" pattern targets = [2, 4, 6] for x in targets: if x % 2 != 0: break else: print("All even!") # runs — no break hit
04 · Functions
defining & calling functions
Functions are defined with def. They support default parameters, keyword arguments, *args (variable positional), and **kwargs (variable keyword). Functions always return a value — omitting return returns None. Always use a docstring to document purpose.
def power(base, exp=2): """Return base raised to exp (default: square).""" return base ** exp print(power(3)) # 9 print(power(2, 10)) # 1024 print(power(exp=3, base=4)) # 64 (keyword args) def summarise(*args, **kwargs): print("positional:", args) print("keyword:", kwargs) summarise(1, 2, name="Glenn", city="Manila")
dynamic functions & lambdas
Functions are first-class objects — assign them to variables, store in lists, pass as arguments. lambda creates a single-expression anonymous function. This enables map(), filter(), sorted(key=...), and decorators.
double = lambda x: x * 2 print(double(5)) # 10 nums = [3, 1, 4, 1, 5, 9] squares = list(map(lambda x: x**2, nums)) evens = list(filter(lambda x: x%2==0, nums)) print(squares) # [9, 1, 16, 1, 25, 81] words = ["banana", "apple", "cherry"] print(sorted(words, key=lambda w: len(w)))
comprehensions
Comprehensions are concise, readable, and faster than equivalent for-loops. Python supports list, dict, set, and generator comprehensions. Pattern: [expr for item in iterable if condition].
# List comprehension squares = [x**2 for x in range(10)] even_sq = [x**2 for x in range(10) if x%2==0] # Dict comprehension word_len = {w: len(w) for w in ["py", "code", "zen"]} # Set comprehension unique_lens = {len(w) for w in ["hi", "hey", "hello"]} # Generator (lazy — no list built in memory) total = sum(x**2 for x in range(1_000_000))
05 · Object-Oriented Programming
defining a class
A class is a blueprint; an object (instance) is a concrete thing built from it. __init__ is the constructor. self is the instance. Define class attributes (shared by all instances) and instance attributes (unique per object). Always implement __repr__ for readable debugging output.
class Dog: species = "Canis familiaris" # class attribute def __init__(self, name, age): self.name = name # instance attributes self.age = age def bark(self): return f"{self.name} says Woof!" def __repr__(self): return f"Dog('{self.name}', {self.age})" rex = Dog("Rex", 3) print(rex.bark()) # Rex says Woof! print(rex) # Dog('Rex', 3)
inheritance & polymorphism
A child class inherits all attributes and methods from its parent. Call the parent's constructor with super().__init__(). Override methods in the child to specialise behaviour. Polymorphism: different classes can respond to the same method name in their own way — enabling flexible, extensible code.
class Animal: def __init__(self, name): self.name = name def speak(self): return "..." class Dog(Animal): def speak(self): return f"{self.name}: Woof!" class Cat(Animal): def speak(self): return f"{self.name}: Meow!" # Polymorphism — same interface, different behaviour for animal in [Dog("Rex"), Cat("Luna")]: print(animal.speak())
encapsulation & properties
Use a single underscore (_name) to signal "internal use" (protected by convention), and double underscore (__name) to trigger name mangling (private). The @property decorator turns a method into a readable attribute, and @attr.setter adds validated write access — the Pythonic way to do getters/setters.
class BankAccount: def __init__(self, balance=0): self.__balance = balance # private @property def balance(self): return self.__balance # read only def deposit(self, amount): if amount <= 0: raise ValueError("Must be positive") self.__balance += amount acc = BankAccount(100) acc.deposit(50) print(acc.balance) # 150
06 · Modules & Packages
modules & import system
Every .py file is a module. Import with import module or from module import name. Packages are folders with __init__.py. The standard library covers math, file I/O, networking, dates, JSON, CSV, regex, and more — no pip needed.
import math print(math.sqrt(144), math.pi) from datetime import date, timedelta today = date.today() next_week = today + timedelta(days=7) print(today, "→", next_week) import json data = {"name": "Glenn", "score": 99} print(json.dumps(data)) # '{"name": "Glenn", "score": 99}'
popular third-party packages
Install with pip install <name>. PyPI has 500,000+ packages. Essential ones:
  • numpy — fast arrays and linear algebra
  • pandas — DataFrames for data analysis (like Excel in Python)
  • matplotlib / seaborn — plotting and visualisation
  • requests — simple HTTP calls and API consumption
  • fastapi / flask — web frameworks for building APIs
  • sqlalchemy — database ORM for SQL databases
  • pytest — powerful testing framework
  • torch / tensorflow — deep learning and AI
07 · Error Handling & File I/O
exception handling
try / except / else / finally manages error flow. except catches specific exceptions (always be specific — avoid bare except:). else runs only if no exception occurred. finally always runs — perfect for cleanup. Raise custom errors by subclassing Exception.
try: value = int(input("Enter a number: ")) result = 10 / value except ValueError: print("Not a valid integer!") except ZeroDivisionError: print("Cannot divide by zero!") else: print(f"Result: {result}") # only if no error finally: print("Done.") # always runs
file I/O
Use open() to read and write files. Always use a with block — it guarantees the file closes even if an error occurs. Mode flags: "r" read, "w" write (overwrites), "a" append, "rb"/"wb" binary. Use pathlib.Path for modern, cross-platform file paths.
# Write with open("notes.txt", "w") as f: f.write("Python is amazing!\n") # Read all at once with open("notes.txt", "r") as f: content = f.read() # Read line by line (memory efficient) with open("notes.txt") as f: for line in f: print(line.strip())
08 · Advanced Patterns
iterators & generators
An iterator implements __iter__() and __next__(). A generator uses yield to produce values one at a time without building the full sequence in memory — perfect for large data, infinite sequences, and streaming pipelines. Generator expressions (sum(x**2 for x in ...)) are even more concise.
def countdown(n): while n > 0: yield n # suspends here, resumes on next call n -= 1 for num in countdown(5): print(num, end=" ") # 5 4 3 2 1 # Infinite Fibonacci generator def fibonacci(): a, b = 0, 1 while True: yield a a, b = b, a + b
decorators
A decorator is a function that wraps another function — adding behaviour (logging, timing, caching, auth) without touching the original code. The @name syntax is sugar for func = decorator(func). The standard library provides @functools.lru_cache, @staticmethod, @classmethod, and @property.
import time from functools import wraps def timer(func): @wraps(func) def wrapper(*args, **kwargs): start = time.perf_counter() result = func(*args, **kwargs) end = time.perf_counter() print(f"{func.__name__} → {end-start:.4f}s") return result return wrapper @timer def slow_sum(n): return sum(range(n)) slow_sum(10_000_000) # slow_sum → 0.2134s
type hints 3.5+
Type hints annotate variables and function signatures with expected types. Python does not enforce them at runtime — they are for IDEs, linters (mypy), and documentation. In Python 3.9+ use built-in generics: list[int], dict[str, Any]. Python 3.10+ uses X | Y union syntax.
def greet(name: str, times: int = 1) -> str: return (f"Hello {name}! ") * times def total(nums: list[int]) -> int: return sum(nums) # Union type — int OR str (Python 3.10+ syntax) def parse(value: int | str) -> float: return float(value)

10 · Pythonic Style & Best Practices
the Zen of Python
Run import this in any Python shell to read all 19 aphorisms. Key ones every beginner should internalise:
  • Beautiful is better than ugly.
  • Explicit is better than implicit.
  • Simple is better than complex.
  • Readability counts.
  • Errors should never pass silently.
  • There should be one obvious way to do it.
  • If the implementation is hard to explain, it's a bad idea.
pythonic idioms to prefer
  • for x in items over for i in range(len(items))
  • f-strings over "Hello " + name
  • with open(...) as f over manual f.close()
  • List comprehensions over append() in a loop
  • x is None over x == None
  • if items: over if len(items) != 0:
  • Unpacking: first, *rest = [1, 2, 3, 4]
  • Dict merge 3.9+: merged = dict_a | dict_b
  • Positional-only params 3.8+: def f(x, /, y): ...
Use python -m py_compile file.py to check syntax, python -m pdb file.py to debug, and python -m venv .venv to create an isolated virtual environment for each project.
