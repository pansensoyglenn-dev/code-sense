01 · What Is Artificial Intelligence?
AI defined
Artificial Intelligence (AI) is the field of computer science focused on building systems that can perform tasks that typically require human intelligence — reasoning, learning, perception, language understanding, and decision-making. Modern AI is largely data-driven: systems learn patterns from examples rather than being explicitly programmed with rules. Python is the dominant language of AI because of its readability, rich ecosystem, and overwhelming adoption in research and industry.
the AI landscape
AI is a broad field with several distinct subfields, each addressing a different class of problem:
  • Machine Learning (ML) — algorithms that learn from data to make predictions or decisions without being explicitly programmed.
  • Deep Learning (DL) — ML using multi-layer neural networks, enabling breakthroughs in vision, speech, and language.
  • Natural Language Processing (NLP) — teaching machines to understand, generate, and translate human language.
  • Computer Vision (CV) — enabling machines to interpret and understand images and video streams.
  • Reinforcement Learning (RL) — agents learning optimal behaviour by interacting with an environment and receiving reward signals.
  • Generative AI — models that create new content including text, images, audio, code, and video.

02 · Python & NumPy — The Numerical Foundation
why NumPy powers AI
NumPy provides fast, vectorised N-dimensional arrays backed by optimised C code. Every major ML library — PyTorch, TensorFlow, Scikit-learn — operates on NumPy-compatible arrays internally. Vectorised operations avoid slow Python loops, running at near-C speed. Arrays support broadcasting, slicing, reshaping, and full linear algebra operations natively, making them the universal currency of numerical AI computation.

03 · Data Preprocessing
why preprocessing matters
Raw data is almost never ready for a machine learning model. Preprocessing transforms messy, inconsistent input data into a clean, structured format that a model can learn from effectively. The quality of preprocessing directly determines the quality of the model — a well-preprocessed simple model consistently outperforms a complex model fed raw, unclean data. Core steps include handling missing values, encoding categorical variables, scaling numerical features, and splitting data into training and test sets.

04 · Machine Learning Algorithms
linear regression from scratch
Linear regression is the simplest and most fundamental ML algorithm. It models the relationship between input features and a continuous output as a straight line (or hyperplane in higher dimensions). The model finds the weights that minimise the Mean Squared Error between predictions and true values, either by solving the closed-form normal equation or iteratively via gradient descent. Despite its simplicity, linear regression remains one of the most widely used models in production.

05 · Neural Networks from Scratch
how neural networks learn
A neural network consists of layers of neurons, each computing a weighted sum of its inputs followed by a non-linear activation function. The network learns by adjusting its weights through backpropagation — computing the gradient of a loss function with respect to each weight, then taking a small step in the direction that reduces the loss. This iterative process is called gradient descent. Understanding it from scratch builds deep intuition before working with high-level frameworks like PyTorch.

06 · Natural Language Processing
text processing fundamentals
Natural Language Processing (NLP) enables machines to read, understand, and generate human language. Before any model can process text, it must be converted into numerical form. This pipeline involves tokenisation (splitting text into tokens), cleaning (lowercasing, removing punctuation), building a vocabulary, and representing documents as vectors — either using bag-of-words, TF-IDF, or modern contextual embeddings from transformer models. Understanding these foundational representations is essential before using high-level NLP libraries.

07 · Clustering & Unsupervised Learning
discovering structure without labels
Unsupervised learning finds hidden structure in data without any labelled examples. It is used when labelling data is expensive, impractical, or when the goal is exploratory — understanding what natural groupings exist in a dataset. K-Means clustering is the most widely used clustering algorithm: it partitions data into K groups by iteratively assigning points to the nearest centroid and recomputing centroids until convergence. Applications include customer segmentation, document grouping, anomaly detection, and feature learning.

08 · Evaluation & Model Selection
measuring model performance
Choosing the right evaluation metric is as important as choosing the right model. Accuracy is intuitive but misleading on imbalanced datasets — a model that always predicts the majority class can achieve 95% accuracy while being completely useless. Precision measures the fraction of positive predictions that are actually positive. Recall measures the fraction of actual positives that were detected. The F1 score balances both. Cross-validation provides a more reliable performance estimate by evaluating the model on multiple train/test splits.

09 · Generative AI Concepts
how language models work
Language models learn to predict the next token in a sequence given all preceding tokens. Through this simple objective applied to enormous text corpora, they develop the ability to reason, summarise, translate, write code, and answer questions. A transformer model uses the attention mechanism to weigh the relevance of every previous token when predicting the next one. Temperature controls randomness: at temperature 0 the model always picks the most likely token; at higher temperatures output becomes more creative and diverse.

10 · AI Best Practices & Ethics
responsible AI development
Building AI responsibly is as important as building it effectively. Every AI developer must understand these core principles: Fairness — models trained on biased data amplify those biases; audit datasets and evaluate performance across demographic groups. Transparency — document what your model does, its limitations, and its training data. Privacy — protect sensitive data; use differential privacy or synthetic data where appropriate. Robustness — test against distribution shift and edge cases before deployment. Human oversight — keep humans in the loop for high-stakes decisions; AI should assist, not unilaterally decide.
