Object-Oriented Programming in Python — Core Concepts

This guide presents Python’s object model with emphasis on rationale and practical application rather than syntax alone. Each concept is illustrated using a consistent mining and drilling equipment theme to demonstrate real-world utility.

1. Classes & Instances — Blueprints vs. Concrete Objects

A class serves as a blueprint that defines structure and behavior. An instance is a specific object created from that blueprint, with its own unique state. Class attributes are shared across all instances, while instance attributes are unique to each object. This design promotes memory efficiency and enables flexible per-instance customization.

class MiningDrill:
    # Shared across all drills
    category = "rotary"

    def __init__(self, model: str, power_kw: int):
        self.model = model
        self.power_kw = power_kw
        self.is_running = False

    def start(self) -> str:
        self.is_running = True
        return f"{self.model} started — {self.power_kw} kW"

drill1 = MiningDrill("X900", 450)
drill2 = MiningDrill("Z1100", 620)

print(drill1.start())                 # X900 started — 450 kW
print(drill2.category)                # rotary

MiningDrill.category = "hybrid"
print(drill1.category)                # hybrid
print(drill2.category)                # hybrid

The self parameter refers to the current instance and is essential for accessing or modifying instance-specific data.

2. Inheritance — Reuse and Specialization

Inheritance allows a child class to inherit attributes and methods from a parent class while adding or modifying behavior. The super() function provides clean access to parent implementations, supporting the open/closed principle: classes should be open for extension but closed for modification.

class Tool:
    def __init__(self, name: str):
        self.name = name

    def use(self) -> str:
        return f"using {self.name}"

class Drill(Tool):
    def __init__(self, name: str, power_kw: int):
        super().__init__(name)
        self.power_kw = power_kw

    def use(self) -> str:
        parent_use = super().use()
        return parent_use + f" with {self.power_kw} kW power"

hammer_drill = Drill("HammerDrill", 300)
print(hammer_drill.use())

Inheritance enables code reuse, but excessive depth can lead to rigidity. Composition is often preferred when flexibility is more important than an is-a relationship.

🔹 Design tip: Prefer mixins or composition when behavior is shared across unrelated classes. Inheritance excels for incremental refinement.

3. Encapsulation & Properties — Controlled Access

Encapsulation protects internal state by controlling access. Python uses naming conventions (_ for protected, __ for name mangling) rather than strict access modifiers. Properties provide a clean interface for getter/setter logic without changing the public API.

class TemperatureSensor:
    def __init__(self):
        self._celsius = 20.0

    @property
    def celsius(self) -> float:
        return self._celsius

    @celsius.setter
    def celsius(self, value: float):
        if not (-50 <= value <= 150):
            raise ValueError("Temperature outside safe range")
        self._celsius = value

    @property
    def fahrenheit(self) -> float:
        return self._celsius * 9/5 + 32

sensor = TemperatureSensor()
sensor.celsius = 85.0
print(sensor.celsius)      # 85.0
print(sensor.fahrenheit)   # 185.0

Properties preserve a simple attribute-like syntax while allowing validation, computation, or lazy evaluation.

4. Polymorphism & Duck Typing — Behavior over Explicit Type

Polymorphism allows objects of different classes to be treated uniformly if they implement the expected interface. Python relies on duck typing: if an object has the required methods, it is accepted — no formal interface declaration is needed.

class Drill:
    def action(self): return "drilling rock"

class BlastingTeam:
    def action(self): return "setting explosives"

class Hauler:
    def action(self): return "transporting ore"

def perform_actions(equipment):
    for eq in equipment:
        print(eq.action())

team = [Drill(), BlastingTeam(), Hauler()]
perform_actions(team)

Duck typing provides exceptional flexibility and is the foundation of many Python idioms and standard library functions.

5. Magic Methods (Dunder Methods) — Pythonic Behavior

Magic methods allow custom classes to integrate naturally with Python syntax and built-in operations (len(), str(), operators, etc.).

class OreVein:
    def __init__(self, mineral: str, grade: float):
        self.mineral = mineral
        self.grade = grade

    def __str__(self) -> str:
        return f"{self.mineral} ({self.grade}% grade)"

    def __repr__(self) -> str:
        return f"OreVein('{self.mineral}', {self.grade})"

    def __len__(self) -> int:
        return int(self.grade * 10)

    def __lt__(self, other) -> bool:
        return self.grade < other.grade

    def __add__(self, other):
        avg = (self.grade + other.grade) / 2
        return OreVein("mixed", avg)

v1 = OreVein("gold", 5.2)
v2 = OreVein("silver", 3.8)

print(v1)              # gold (5.2% grade)
print(repr(v1))        # OreVein('gold', 5.2)
print(len(v1))         # 52
print(v1 > v2)         # True
print(v1 + v2)         # OreVein('mixed', 4.5)

6. Class Methods and Static Methods — Alternative Perspectives

Class methods receive the class as the first argument (cls) and are often used for alternative constructors. Static methods receive no implicit first argument and serve as utility functions logically grouped with the class.

class Survey:
    total_surveys = 0

    def __init__(self, area: str):
        self.area = area
        Survey.total_surveys += 1

    @classmethod
    def from_string(cls, data: str):
        area = data.split(":")[0]
        return cls(area)

    @classmethod
    def count(cls) -> str:
        return f"Total surveys: {cls.total_surveys}"

    @staticmethod
    def is_valid_area(name: str) -> bool:
        cleaned = name.replace("_", "")
        return len(name) > 3 and cleaned.isalnum()

s1 = Survey("NorthPit")
s2 = Survey.from_string("SouthPit:gold")
print(Survey.count())
print(Survey.is_valid_area("West_Zone"))

7. Composition — Flexible Object Construction

Composition builds complex objects by combining simpler ones (“has-a” relationship), offering greater flexibility and testability than deep inheritance hierarchies.

class Engine:
    def start(self): return "engine running"
    def stop(self): return "engine stopped"

class DrillBit:
    def __init__(self, material: str):
        self.material = material

    def spin(self):
        return f"{self.material} bit spinning"

class DrillingRig:
    def __init__(self, bit_material: str):
        self.engine = Engine()
        self.bit = DrillBit(bit_material)

    def start_operation(self):
        return f"{self.engine.start()} — {self.bit.spin()}"

    def stop_operation(self):
        return self.engine.stop()

rig = DrillingRig("diamond")
print(rig.start_operation())
print(rig.stop_operation())

8. Abstract Base Classes — Enforcing Interfaces

Abstract base classes (ABCs) define a contract that concrete subclasses must fulfill. Attempting to instantiate an abstract class or a subclass missing required methods raises an error.

from abc import ABC, abstractmethod

class MiningEquipment(ABC):
    def __init__(self, name: str):
        self.name = name

    @abstractmethod
    def operate(self) -> str:
        pass

    def info(self) -> str:
        return f"Equipment: {self.name}"

class Excavator(MiningEquipment):
    def operate(self) -> str:
        return f"{self.name} is excavating"

e = Excavator("CAT 320")
print(e.operate())
print(e.info())

9. Dataclasses — Reduced Boilerplate for Data Containers

Introduced in Python 3.7, dataclasses automatically generate __init__, __repr__, __eq__, and other methods, making them ideal for data-oriented classes.

from dataclasses import dataclass, field

@dataclass
class MineralDeposit:
    name: str
    depth: float
    grade: float
    active: bool = True
    tags: list[str] = field(default_factory=list)

    def extractable(self) -> bool:
        return self.grade > 1.5 and self.depth < 1000

d1 = MineralDeposit("copper", 350.2, 2.3)
d2 = MineralDeposit("gold", 820.0, 5.1, tags=["high-grade"])

print(d1)
print(d1.extractable())
print(d1 == d2)

10. Builder Pattern — Fluent Complex Object Construction

The builder pattern provides a readable, chainable interface for constructing objects with many optional parameters.

class DrillConfiguration:
    def __init__(self, model: str, power_kw: int):
        self.model = model
        self.power_kw = power_kw
        self.depth_capacity = 1000
        self.bit_type = "standard"
        self.automatic_lubrication = False

class DrillBuilder:
    def __init__(self, model: str, power_kw: int):
        self._config = DrillConfiguration(model, power_kw)

    def set_depth_capacity(self, depth: int):
        self._config.depth_capacity = depth
        return self

    def set_bit_type(self, bit: str):
        self._config.bit_type = bit
        return self

    def enable_autolube(self):
        self._config.automatic_lubrication = True
        return self

    def build(self):
        return self._config

config = (DrillBuilder("MegaDrill", 800)
          .set_depth_capacity(2500)
          .set_bit_type("diamond")
          .enable_autolube()
          .build())

print(config.bit_type)
print(config.automatic_lubrication)
