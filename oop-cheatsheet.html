📦 class & instance blueprint

`__init__` / `self` / attributes
class Employee: company = "OpenAI" # class variable def __init__(self, name, pay): self.name = name # instance variable self.pay = pay def raise_pay(self, factor): self.pay *= factor e = Employee("Alice", 90000) e.raise_pay(1.1)
  • self – reference to current instance
  • class var – shared across instances
  • instance var – unique per object

🧬 inheritance is-a

`super()` / MRO / method override
class Vehicle: def move(self): return "moving.." class Car(Vehicle): def __init__(self, brand): self.brand = brand def move(self): # override return f"{self.brand} drives" c = Car("Tesla") print(c.move()) # Tesla drives
  • super() – call parent method

🔒 encapsulation name mangling

_protected / __private / properties
class BankAccount: def __init__(self, owner, balance): self.owner = owner self.__balance = balance # private @property def balance(self): return self.__balance def withdraw(self, amount): if amount <= self.__balance: self.__balance -= amount acc = BankAccount("Bob", 1000) # acc.__balance → error (mangled to _BankAccount__balance)
  • _single – "protected" (convention)
  • __double – name mangling
  • @property – controlled access

🎭 polymorphism many forms

duck typing / protocols / ABC
from abc import ABC, abstractmethod class Animal(ABC): @abstractmethod def speak(self): pass class Dog(Animal): def speak(self): return "woof" class Cat(Animal): def speak(self): return "meow" def make_sound(animal): print(animal.speak()) # works for any Animal
  • duck typing – "if it walks like a duck"
  • ABC – enforce interface
  • @abstractmethod – must implement

⚙️ dunder methods __repr__, __len__ …

operator overloading, built‑ins
class Book: def __init__(self, title, pages): self.title = title self.pages = pages def __repr__(self): return f"Book({self.title})" def __len__(self): return self.pages def __eq__(self, other): return self.title == other.title b = Book("1984", 328) print(repr(b)) # Book(1984) print(len(b)) # 328
  • __str__ / __repr__ – string representation
  • __add__, __len__, __getitem__
  • __call__ – object as function

🏷️ class & static methods

@classmethod / @staticmethod
class Date: def __init__(self, year, month, day): self.year = year self.month = month self.day = day @classmethod def from_string(cls, s): # factory y, m, d = map(int, s.split('-')) return cls(y, m, d) @staticmethod def is_valid(s): # utility y, m, d = map(int, s.split('-')) return 1 <= m <= 12 and 1 <= d <= 31
  • @classmethod – receives class (cls)
  • @staticmethod – no self/cls
  • used for alternative constructors, helpers

🔗 composition & aggregation

"has‑a" vs "owns"
class Engine: pass class Car: def __init__(self, engine): # composition (engine injected) self.engine = engine # strong ownership class Wheel: pass class CarAssembly: def __init__(self): self.wheels = [Wheel() for _ in range(4)] # composition # or aggregation: wheels passed from outside
  • composition – part cannot exist without whole
  • aggregation – weaker, shared references
  • favour composition over inheritance

🧩 singleton (metaclass)

class SingletonMeta(type): _instances = {} def __call__(cls, *args, **kwargs): if cls not in cls._instances: cls._instances[cls] = super().__call__(*args, **kwargs) return cls._instances[cls] class Logger(metaclass=SingletonMeta): pass # every Logger() returns same instance

class ManagedFile: def __init__(self, name): self.name = name def __enter__(self): self.file = open(self.name, 'w') return self.file def __exit__(self, *args): self.file.close() # with ManagedFile('hello.txt') as f: # f.write('hi')
