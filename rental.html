INTRODUCTION

What Is rental.py and Why Does It Matter?

Picture the sun-drenched shores of El Nido, Palawan — one of the most visited destinations in the Philippines. Tourists arrive daily wanting to explore hidden lagoons by boat or zip through mountain roads on rented motorcycles. Every booking requires the same foundation: collect the customer's name and phone number, confirm payment, and issue a receipt. rental.py is the Python system that handles all of it from the command line.

The program's architecture is a clean demonstration of inheritance. Both services — motorcycle rental and boat booking — need customer registration, payment processing, and receipt generation. Rather than duplicating those three methods in every service class, they are written once in a parent class called BaseService, and both MotorcycleRental and BoatBooking inherit them automatically. Child classes only add what is uniquely theirs: MotorcycleRental adds a fleet dictionary and booking logic; BoatBooking adds hotel pickup input.

This guide walks through every class and every method in order — explaining not just what the code does but why each pattern was chosen. You will understand class variables versus instance variables, regex phone validation, file I/O for receipt generation, random reference code creation, and the if __name__ == "__main__" guard, all through one cohesive, runnable program set in a place you can picture.

🏝️
What you will learn: Inheritance and shared parent methods, class variables vs instance variables, re.match() for Philippine phone number validation, the while True / break input validation loop pattern, random.randint() for reference code generation, file I/O with open() and write() for receipt saving, f-string formatting for currency, and the if __name__ == "__main__" entry guard.

SECTION 03

The BaseService Class

class BaseService — __init__
class BaseService:
    def __init__(self):
        self.first_name = ""
        self.last_name  = ""
        self.full_name  = ""
        self.phone_number = ""

BaseService is the parent class of the entire system. Its __init__ method initialises four instance variables — all empty strings — that will be populated later by get_customer_info(). Initialising attributes in __init__ even when they start empty is a best practice: it declares upfront exactly what data every BaseService object (and by inheritance, every child object) will carry. Any method that accesses self.full_name can do so safely because the attribute is guaranteed to exist.

Notice that BaseService.__init__ takes no parameters beyond self. This is intentional — customer details are not known at object creation time. They are collected interactively later, which is why they are set to empty strings first and filled in by get_customer_info() during the booking flow.

🔑
Key insight: Because MotorcycleRental and BoatBooking inherit from BaseService without defining their own __init__, Python automatically calls BaseService.__init__(self) when either child class is instantiated. The four empty string attributes are set on every booking object for free.
SECTION 04

get_customer_info() — Regex Phone Validation

get_customer_info()
    def get_customer_info(self):
        while True:
            self.first_name = input("Enter First Name: ").strip().capitalize()
            self.last_name  = input("Enter Last Name:  ").strip().capitalize()
            if self.first_name and self.last_name:
                break
            print("❌ Error: Both names are required.")

        while True:
            print("Format: +639XXXXXXXXX")
            self.phone_number = input("Enter Phone Number: ").strip()
            if re.match(r'^\+639\d{9}$', self.phone_number):
                break
            print("❌ Error: Invalid format. Please use +639XXXXXXXXX.")

        self.full_name = f"{self.first_name} {self.last_name}"
        return True

This method uses two separate while True loops — one for the name and one for the phone number. Each loop keeps asking until valid input is received, then breaks. This is the standard Python pattern for input validation: loop forever, break on success. The name loop checks that both first_name and last_name are non-empty strings (Python treats an empty string as falsy). The .strip().capitalize() chain removes leading/trailing whitespace and ensures the first letter is uppercase.

Understanding the Regex Pattern

The pattern r'^\+639\d{9}$' validates Philippine mobile numbers in international format:

TokenMeaningExample Match
^Must start here — no characters before +639+639…
\+Literal plus sign (escaped because + has special meaning in regex)+
639Philippine country code + mobile prefix639
\d{9}Exactly 9 decimal digits123456789
$Must end here — no extra characters after the 9 digitsend

A valid number looks like +639171234567. The anchors ^ and $ ensure the pattern matches the entire string — without them, +639171234567extra would also pass because the prefix matches. re.match() returns a match object (truthy) on success or None (falsy) on failure, which is why it works directly in the if condition.

SECTION 05

process_payment() — PayPal Simulation

process_payment()
    def process_payment(self, amount: float, service_name: str):
        print(f"💳 SECURE CHECKOUT: {service_name}")
        print(f"Total Due: PHP {amount:,.2f}")

        while True:
            confirm = input("Confirm payment via PayPal? (Y/N): ").strip().upper()
            if confirm == 'Y':
                return f"ELNIDO-{random.randint(100000, 999999)}"
            if confirm == 'N':
                return None
            print("❌ Please enter 'Y' or 'N'.")

This method accepts the total amount and the service_name as parameters, displays the checkout summary, and enters a while True confirmation loop. Calling .upper() on the input normalises both y and Y to 'Y', making the comparison case-insensitive without extra logic.

On confirmation, random.randint(100000, 999999) generates a 6-digit integer and embeds it in the reference code string "ELNIDO-XXXXXX". This reference code is returned to the caller — it becomes the filename for the receipt and the identifier printed on it. On cancellation, the method returns None, which the calling code checks with if token: to skip receipt generation.

🎲
Return value design: process_payment() returns either a reference code string or None. This two-outcome return value is a clean Python pattern — the caller decides what to do on each outcome without needing to catch exceptions. In a real system, this method would call a payment API and return the transaction ID from the provider.
SECTION 06

generate_receipt() — File I/O

generate_receipt()
    def generate_receipt(self, service_details, total_amount, reference_code):
        filename = f"Receipt_{reference_code}.txt"
        receipt_content = f"""
==========================================
       EL NIDO RENTAL SERVICES
       Official Booking Receipt
==========================================
Reference Code: {reference_code}
Customer:       {self.full_name}
Phone:          {self.phone_number}
------------------------------------------
Service Details:
{service_details}
------------------------------------------
TOTAL PAID:     PHP {total_amount:,.2f}
Status:         PAID (via PayPal)
------------------------------------------
Thank you for booking with us!
==========================================
"""
        with open(filename, "w", encoding="utf-8") as file:
            file.write(receipt_content)
        print(f"✅ SUCCESS! Receipt saved as: {filename}")

This method constructs the receipt as a multi-line f-string and writes it to a .txt file. The filename is derived from the reference code — Receipt_ELNIDO-123456.txt — so every booking produces a uniquely named file. The f-string uses self.full_name and self.phone_number, which are instance variables set earlier by get_customer_info(). The service_details parameter is passed in by the child class, allowing the same receipt template to serve both motorcycle and boat bookings.

The with open(filename, "w", encoding="utf-8") as file: context manager is the correct Python pattern for file writing. The with block guarantees the file is properly closed after writing — even if an exception occurs mid-write. encoding="utf-8" ensures that any Unicode characters (like the ₱ peso symbol, if added later) write correctly on all operating systems.

📁
Where is the file saved? The file is created in the current working directory — wherever you run python rental.py from. On Windows this might be C:\Users\YourName\, on macOS/Linux it will be the terminal's active directory. Each booking creates one new Receipt_ELNIDO-XXXXXX.txt file.
SECTION 07

The MotorcycleRental Class

class MotorcycleRental
class MotorcycleRental(BaseService):
    FLEET = {
        1: {"model": "Honda Click 125",    "rate": 500},
        2: {"model": "Yamaha Mio",          "rate": 800},
        3: {"model": "Suzuki Raider",       "rate": 1200},
        4: {"model": "Kawasaki Ninja 400",  "rate": 2000}
    }

    def run_booking(self):
        for k, v in self.FLEET.items():
            print(f"{k}. {v['model']} - PHP {v['rate']}/day")

        while True:
            try:
                choice = int(input("Select Motorcycle (1-4): "))
                if choice not in self.FLEET:
                    print(f"❌ '{choice}' is not available.")
                    continue
                days = int(input("Number of days: "))
                if days < 1:
                    print("❌ Minimum rental is 1 day.")
                    continue
                break
            except ValueError:
                print("❌ Invalid input. Please enter numbers only.")

        selected = self.FLEET[choice]
        total = selected['rate'] * days

        if self.get_customer_info():
            token = self.process_payment(total, selected['model'])
            if token:
                details = f"Rental Item: {selected['model']}\nDuration:    {days} Day(s)"
                self.generate_receipt(details, total, token)

FLEET is a class variable — defined directly in the class body, outside any method. This means it is shared across all instances of MotorcycleRental. There is no need to create a new copy of the fleet for every booking object, because the fleet options never change between bookings. This is the key distinction between class variables (shared, defined on the class) and instance variables (per-object, set in __init__).

The run_booking() method wraps the selection and day-count inputs in a single try/except ValueError block. If the user types "abc", int() raises a ValueError which is caught and prints an error before looping again. The choice not in self.FLEET check validates the range. Once both inputs are valid, the total is calculated as rate × days, and the three inherited methods are called in sequence.

SECTION 08

The BoatBooking Class

class BoatBooking
class BoatBooking(BaseService):
    def run_booking(self):
        print("\n⛵ PRIVATE BOAT CHARTER")
        if self.get_customer_info():
            hotel = input("Enter Hotel Name for Pickup: ").strip()
            total = 5000.00
            token = self.process_payment(total, "Boat Tour")
            if token:
                details = f"Service:     Private Boat Charter\nPickup Loc:  {hotel}"
                self.generate_receipt(details, total, token)

BoatBooking is the simplest child class in the program — just 8 lines of actual logic. There is no fleet to select from, no day calculation, and no dynamic pricing. The service is a fixed private boat charter at PHP 5,000. The only unique input is the hotel name for pickup, which gets embedded in the receipt's service details.

This simplicity illustrates an important design principle: child classes only need to define what is unique to them. Everything else — customer registration, payment, receipt generation — is handled by the parent. BoatBooking adds exactly one unique piece of data (hotel name) and one unique piece of logic (fixed price), and inherits all the rest.

⚖️
MotorcycleRental vs BoatBooking: Both child classes call the same three inherited methods in the same order — get_customer_info(), then process_payment(), then generate_receipt(). The difference is entirely in what happens before those calls: motorcycle builds a fleet menu and calculates a variable total; boat collects a hotel name and uses a fixed total. This is polymorphism at its most practical.
SECTION 09

main() — The Menu Loop

main()
def main():
    while True:
        print("═" * 45)
        print(f"{'EL NIDO BOOKING & MANAGEMENT SYSTEM':^45}")
        print("═" * 45)
        print("1. Rent a Motorcycle")
        print("2. Book a Boat")
        print("3. Exit System")

        choice = input("\nSelect Option: ").strip()
        if choice == "1":
            MotorcycleRental().run_booking()
        elif choice == "2":
            BoatBooking().run_booking()
        elif choice == "3":
            print("System closing. Maraming Salamat!")
            break
        else:
            print("❌ Invalid selection.")

The main() function is a while True loop that re-displays the menu after every completed booking, allowing multiple transactions in a single session. The {....:^45} f-string format specifier centres the title text within 45 characters — a neat formatting trick for terminal output.

Notice the instantiation pattern: MotorcycleRental().run_booking() creates a fresh object and immediately calls run_booking() on it — all in one line. This means every booking gets a clean object with empty instance variables, so customer data from one booking never leaks into the next. The object is discarded after run_booking() returns.

SECTION 10

if __name__ == "__main__" — The Entry Guard

if __name__ == "__main__":
    main()

This two-line guard ensures main() — with all its interactive input() calls and menu printing — only runs when you execute the script directly (python rental.py). If another developer imports BaseService, MotorcycleRental, or BoatBooking into their own project, the booking menu does not start automatically. The classes remain importable and testable without triggering the interactive session.

import random
import re
import sys

class BaseService:
    def __init__(self):
        self.first_name = ""
        self.last_name  = ""
        self.full_name  = ""
        self.phone_number = ""

    def get_customer_info(self):
        print("\n" + "─" * 40)
        print("📝 CUSTOMER REGISTRATION")
        print("─" * 40)
        while True:
            self.first_name = input("Enter First Name: ").strip().capitalize()
            self.last_name  = input("Enter Last Name:  ").strip().capitalize()
            if self.first_name and self.last_name:
                break
            print("❌ Error: Both names are required.")
        while True:
            print("Format: +639XXXXXXXXX")
            self.phone_number = input("Enter Phone Number: ").strip()
            if re.match(r'^\+639\d{9}$', self.phone_number):
                break
            print("❌ Error: Invalid format. Please use +639XXXXXXXXX.")
        self.full_name = f"{self.first_name} {self.last_name}"
        return True

    def process_payment(self, amount: float, service_name: str):
        print("\n" + "─" * 40)
        print(f"💳 SECURE CHECKOUT: {service_name}")
        print(f"Total Due: PHP {amount:,.2f}")
        while True:
            confirm = input("Confirm payment via PayPal? (Y/N): ").strip().upper()
            if confirm == 'Y':
                return f"ELNIDO-{random.randint(100000, 999999)}"
            if confirm == 'N':
                return None
            print("❌ Please enter 'Y' or 'N'.")

    def generate_receipt(self, service_details, total_amount, reference_code):

==========================================
